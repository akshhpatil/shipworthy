Capabilities
Everything Happens Invisibly
You keep vibe coding at full speed. Shipworthy handles the engineering you don't know to ask for.
📋

Invisible Specifications

Before code starts, a lightweight spec is generated silently. Deliverables, acceptance criteria, constraints. You never see it. Claude builds against it.

✅

Tests Before Code

TDD happens automatically. Failing test first, then implementation, then verification. You see "it works." Underneath: 12 tests prove it.

🛡

Real-Time Guardrails

5 hooks guard every action. Secrets caught before files are written. Destructive commands blocked before execution. All advisory, never blocking.

📈

Scaling Quality Gates

5 files: light touch. 50 files: coverage thresholds. 500 files: security scans, performance budgets, circular import detection. Automatic.

🔄

Self-Improving Loop

Run /retro after a session. Sub-agents analyze corrections, patterns, and gaps. Learnings persist. Next session starts smarter. Eventually: one-shot.

🏢

Enterprise Ready

PII detection, container security, API versioning, zero-downtime migrations, code complexity limits. Per-project configuration and preset bundles.

Under The Hood
What Happens When You Say
"Build Me an Invoice App"
1

Session Starts

Hook fires in <2s. Detects project type, maturity tier, diagnoses gaps, loads architecture spec and learnings from previous retrospectives.

2

Intent to Spec

Generates a lightweight specification silently. What you want, deliverables, acceptance criteria. For non-technical users: completely invisible.

3

Architecture Awareness

Detects tech stack, generates architecture spec with mandatory rules. Persists across sessions. Claude remembers your project structure tomorrow.

4

Test-Driven Development

Writes failing test. Implements minimal code to pass. Verifies. Installs validation libraries, structured logging, real database. All automatic.

5

Verification

Runs proof commands: tests pass, linter clean, build succeeds. Not "should work" — proven with evidence from actual command output.

6

Retrospective

Run /retro to extract session learnings. Corrections, patterns, and gaps are analyzed by sub-agents. Approved learnings persist for next session.
