How it works
Install once. Everything
else is automatic.

Run one command. Shipworthy installs as a Claude Code plugin. Hooks monitor every file write and bash command in real time. Skills activate on demand based on what you're building. No configuration files to edit, no workflows to learn.

You code normally. Say "build me a payment API" and Shipworthy automatically applies TDD, input validation, secrets scanning, and architecture decisions underneath. Every action is logged transparently to your terminal. Nothing is hidden.

Quality compounds. An architecture spec persists across sessions — session 5 remembers what session 1 decided. Run /retro to extract learnings. The plugin improves with your project.

What's included
55 skills across 11 categories.

Each skill is a Markdown file with YAML frontmatter. No code. No runtime. No dependencies. Just instructions that make AI agents better engineers.

Security, testing, architecture, operations, planning, debugging, frontend, collaboration, documentation — skills activate when relevant and stay silent when they're not.

Transparent hooks

Every file write is scanned for hardcoded secrets. Every bash command is checked for destructive patterns. Everything is logged to your terminal in real time. Six hooks, always watching.

Test-driven by default

Failing test first, then implementation, then verification. TDD happens automatically — not a workflow you have to remember. You see "it works." Underneath: tests prove it.

Adaptive quality gates

5 files: light touch. 50 files: coverage thresholds and lint enforcement. 500 files: security scans, performance budgets, circular import detection. The rigor scales with your project.

11 security profiles

Auto-detects your application type — web app, API, GraphQL, mobile, CLI, data pipeline, IoT, desktop, infrastructure — and applies the right security checks. PII detection, container hardening, OWASP, supply chain.

Architecture as memory

Generates a persistent architecture spec on first run. Directory conventions, naming patterns, mandatory rules. Session 5 knows everything session 1 decided. No more "Claude forgot we use Prisma."
